# Antigravity 작업 지시서

## 한 줄 요약
이 폴더의 슬라이드 이미지들을 사용해 **바이브코딩 사내 교육 플랫폼**(Next.js 14 App Router + TypeScript + Tailwind CSS)을 만들어 주세요. 완성 후 GitHub → Vercel로 배포할 수 있는 상태여야 합니다.

## 작업 순서

1. **`SPEC.md` 를 먼저 정독합니다.** (기술 스택, 디자인 시스템, UI 구조)
2. **`CONTENT.md` 를 정독합니다.** (챕터/레슨 데이터 — 그대로 코드의 데이터 모델로 옮기면 됩니다)
3. `public/slides/part1/`, `public/slides/part2/` 안의 JPG 파일이 강의 본문 이미지입니다. **수정·재인코딩하지 말고 그대로 사용**하세요.
4. `npm create next-app@latest . --ts --tailwind --app --src-dir --eslint --no-import-alias` 로 프로젝트를 초기화하고, SPEC에 정의된 구조로 코드를 작성합니다.
5. 빌드(`npm run build`)가 에러 없이 통과해야 합니다.
6. 완성되면 사용자에게 안내할 수 있도록 `README.md` 를 작성합니다 (로컬 실행 / GitHub 푸시 / Vercel 배포 3-step).

## 절대 지켜야 할 원칙

- **이미지 중심 UI**: 모든 레슨 페이지의 주인공은 슬라이드 이미지입니다. 텍스트 설명은 보조입니다.
- **모던/기술적 톤**: 다크 테마 기본, 모노스페이스 폰트는 코드/명령어에만, 굵은 타이포그래피, 충분한 여백.
- **반응형**: 모바일에서도 슬라이드가 잘 보여야 합니다.
- **접근성**: 모든 이미지에 `alt` 속성, 키보드 네비게이션 가능.
- **외부 라이브러리 최소화**: Tailwind만으로 충분합니다. 추가는 꼭 필요할 때만.
- **CONTENT.md 의 데이터를 그대로 사용**: 챕터/레슨 제목, 설명, 슬라이드 매핑은 임의로 바꾸지 마세요. (오타·문법 수정은 OK)

## 산출물 체크리스트

- [ ] `npm run dev` 로 정상 동작 (포트 3000)
- [ ] `npm run build` 통과
- [ ] 메인 페이지(/) — 챕터 목록 + 진행률 표시
- [ ] 챕터 페이지(/chapters/[chapter-slug]) — 해당 챕터의 레슨 리스트
- [ ] 레슨 페이지(/chapters/[chapter-slug]/[lesson-slug]) — 슬라이드 뷰어 + 설명 + 이전/다음 네비
- [ ] 다크 모드 기본, 라이트 모드 토글 (선택)
- [ ] README.md (로컬 실행, GitHub 푸시, Vercel 배포 가이드 포함)
- [ ] `.gitignore` (node_modules, .next 등)

## 우선순위
**기능 동작 > 디자인 완성도 > 추가 기능**.
사이드바·네비게이션·슬라이드 이미지가 보이는 것이 최우선. 멋진 애니메이션이나 검색 기능은 나중에.
